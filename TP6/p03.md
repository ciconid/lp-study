## Pregunta 3
> Considere el siguiente programa incompleto. Escriba un código para `p` de tal forma que lo que se muestra en la instrucción `write(z)` sea diferente según el pasaje de parámetros sea por valor, por valor-resultado, o por referencia. Indique además cuáles serían los valores mostrados en cada caso.
>
> ```pascal
> program llenar;
> var z: integer;
> procedure p(x: integer)
>   { cuerpo p }
> begin
>   z := 1;
>   p(z);
>   write(z)
> end.
> ```

---

## Respuesta

### Diseño del cuerpo de `p`

Para que `write(z)` muestre un valor distinto bajo cada mecanismo de pasaje, se debe aprovechar el hecho de que —cuando el parámetro real y una variable global se modifican dentro de `p`— los tres modos producen efectos distintos sobre `z` al retornar.

El cuerpo propuesto es:

```pascal
procedure p(x: integer);
begin
  x := 2;
  z := 3;
  x := x + z;
end;
```

---

### Análisis por mecanismo

#### 1. Pasaje por valor (*pass-by-value*)

Cuando un parámetro se pasa por valor, el parámetro formal recibe una **copia** del parámetro real y actúa como variable local; cualquier modificación a `x` no afecta a `z`.

> *"When a parameter is passed by value, the value of the actual parameter is used to initialize the corresponding formal parameter, which then acts as a local variable in the subprogram."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 625)

**Traza de ejecución:**

| Paso | `x` (local) | `z` (global) |
|------|-------------|--------------|
| Llamada `p(z)` con `z = 1` | `x ← 1` (copia) | `z = 1` |
| `x := 2` | `x = 2` | `z = 1` |
| `z := 3` | `x = 2` | `z = 3` |
| `x := x + z` → `x = 2 + 3 = 5` | `x = 5` | `z = 3` |
| Retorno (copia **no** se devuelve) | — | `z = 3` |

**`write(z)` imprime: `3`**

---

#### 2. Pasaje por valor-resultado (*pass-by-value-result*)

El parámetro formal recibe una copia al momento de la llamada (igual que por valor), pero al finalizar la subrutina, el valor final del formal **se copia de vuelta** al parámetro real.

> *"Pass-by-value-result is an implementation model for inout-mode parameters in which actual values are copied. [...] The value of the actual parameter is used to initialize the corresponding formal parameter [...]. At subprogram termination, the value of the formal parameter is transmitted back to the actual parameter."*
> — Sebesta, *Concepts of Programming Languages* E12 (pp. 627–628)

**Traza de ejecución:**

| Paso | `x` (local) | `z` (global) |
|------|-------------|--------------|
| Llamada `p(z)` con `z = 1` | `x ← 1` (copia) | `z = 1` |
| `x := 2` | `x = 2` | `z = 1` |
| `z := 3` | `x = 2` | `z = 3` |
| `x := x + z` → `x = 2 + 3 = 5` | `x = 5` | `z = 3` |
| Retorno: `z ← x` (copy-out) | — | `z = 5` |

**`write(z)` imprime: `5`**

> La diferencia entre valor-resultado y referencia se manifiesta precisamente en casos como este, donde `z` (el parámetro real) es modificada **directamente** dentro del cuerpo de `p`. En valor-resultado, la asignación `z := 3` afecta la variable global, pero al retornar esa modificación es **sobreescrita** por la copia de vuelta de `x`.
> — Sebesta, *Concepts of Programming Languages* E12 (p. 628)

---

#### 3. Pasaje por referencia (*pass-by-reference*)

El parámetro formal recibe el **acceso directo** (dirección) al parámetro real. Toda modificación de `x` dentro de `p` modifica directamente `z`.

> *"Pass-by-reference [...] transmits an access path, usually just an address, to the called subprogram. This provides the access path to the cell storing the actual parameter. Thus, the called subprogram is allowed to access the actual parameter in the calling program unit."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 628)

**Traza de ejecución** (`x` es alias de `z`):

| Paso | `x` (alias de `z`) | `z` (global) |
|------|---------------------|--------------|
| Llamada `p(z)` con `z = 1` | `x` apunta a `z` | `z = 1` |
| `x := 2` | `z = 2` (son la misma celda) | `z = 2` |
| `z := 3` | `z = 3` (y por ende `x = 3`) | `z = 3` |
| `x := x + z` → `x = 3 + 3 = 6` | `z = 6` | `z = 6` |
| Retorno | — | `z = 6` |

**`write(z)` imprime: `6`**

---

### Resumen comparativo

| Mecanismo | Interacción `x` ↔ `z` | `write(z)` |
|-----------|----------------------|------------|
| Por valor | `x` es copia local; cambios a `x` no afectan `z` | **3** |
| Por valor-resultado | `x` es copia local, pero al retornar `z ← x` | **5** |
| Por referencia | `x` es alias directo de `z` | **6** |

Los tres resultados son distintos porque la combinación de modificar **tanto `x` como `z`** dentro de `p` crea una divergencia observable: el pasaje por referencia hace que cada cambio a `x` sea inmediatamente visible en `z` (y viceversa), el pasaje por valor-resultado "aplana" los cambios a `z` al retornar con el último valor de `x`, y el pasaje por valor deja que la asignación directa `z := 3` sea el único cambio visible al exterior.
