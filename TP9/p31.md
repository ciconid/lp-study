## Pregunta 31
> Considere las siguientes expresiones de tipos de funciones en SML. Determine si las funciones son: (1) función curried; (2) forma funcional; (3) función polimórfica.
> a. `f1: ‘a list ->‘b list ->int list`
> b. `f2: (int * int * bool) ->int`

### Respuesta

#### a. `f1: ‘a list ->‘b list ->int list`
- **Función curried:** Sí, porque recibe un argumento y retorna una función que recibe el siguiente argumento.
- **Forma funcional:** No necesariamente, a menos que tome o retorne una función (no se deduce solo del tipo).
- **Función polimórfica:** Sí, porque tiene variables de tipo `'a` y `'b`.


#### b. `f2: (int * int * bool) ->int`
- **Función curried:** No, recibe una tupla como único argumento.
- **Forma funcional:** No, no toma ni retorna funciones.
- **Función polimórfica:** No, solo opera sobre tipos concretos.

#### c. `f3: ‘a list * (‘a * ‘a ->‘a list) * ‘b list * (‘b * ‘b ->‘b list) ->int`
- **Función curried:** No, recibe una tupla como único argumento.
- **Forma funcional:** Sí, porque toma funciones como argumentos (`‘a * ‘a ->‘a list` y `‘b * ‘b ->‘b list`).
- **Función polimórfica:** Sí, porque tiene variables de tipo `'a` y `'b`.

#### d. `f4: int list ->(int ->int) ->int`
- **Función curried:** Sí, recibe un argumento y retorna una función que recibe el siguiente argumento.
- **Forma funcional:** Sí, porque toma una función como argumento (`int -> int`).
- **Función polimórfica:** No, solo opera sobre tipos concretos (`int`).

> *"Una función curried es aquella que recibe sus argumentos de a uno, retornando funciones intermedias. Una función polimórfica es aquella cuyo tipo contiene variables de tipo. Una forma funcional es una función que toma o retorna funciones."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 35, 46)
