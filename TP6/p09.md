## Pregunta 9
> ¿Qué restricciones debe imponer el mecanismo de llamada en un lenguaje que permite parámetros actuales con palabra clave (por nombre)?

---

## Respuesta

### Parámetros con palabra clave (*keyword parameters*)

Los lenguajes de programación suelen asociar parámetros actuales con parámetros formales **por posición** (*positional parameters*): el primer argumento se enlaza al primer parámetro formal, el segundo al segundo, etc. Sin embargo, cuando las listas de parámetros son largas, este mecanismo es propenso a errores de orden.

Como alternativa, algunos lenguajes permiten **parámetros con palabra clave** (*keyword parameters*), donde el nombre del parámetro formal se especifica explícitamente en la llamada:

```python
sumer(length = my_length, list = my_array, sum = my_sum)
```

> *"One solution to this problem is to provide keyword parameters, in which the name of the formal parameter to which an actual parameter is to be bound is specified with the actual parameter in a call. The advantage of keyword parameters is that they can appear in any order in the actual parameter list."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 612)

---

### Restricción principal que debe imponer el mecanismo de llamada

La restricción fundamental es la siguiente:

> **Una vez que aparece un parámetro con palabra clave en la lista de argumentos, todos los parámetros restantes también deben ser con palabra clave.**

Esto se debe a que, al mezclar parámetros posicionales y por nombre, la **posición ya no está bien definida** para los argumentos que siguen a uno con nombre. Si un argumento posicional apareciese después de uno con nombre, el lenguaje no podría determinar de forma inequívoca a qué parámetro formal corresponde.

> *"The only restriction with this approach is that after a keyword parameter appears in the list, all remaining parameters must be keyworded. This restriction is necessary because a position may no longer be well defined after a keyword parameter has appeared."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 613)

**Ejemplo en Python:**

```python
# Válido: primero positional, luego keyword
pay = compute_pay(20000.0, tax_rate=0.15)

# Inválido: keyword seguido de positional (ambigüedad de posición)
# pay = compute_pay(income=20000.0, 0.15)  # ERROR
```

---

### Restricciones adicionales

Además de la restricción de orden, el mecanismo de llamada también debe garantizar:

1. **Que el nombre usado en la llamada coincida con un parámetro formal definido**: el compilador/intérprete debe verificar que el identificador especificado exista en la firma del subprograma.
2. **Que cada parámetro formal sea enlazado exactamente una vez**: no se puede pasar el mismo argumento tanto en forma posicional como por nombre, pues resultaría en una asignación duplicada.

> *"The disadvantage to keyword parameters is that the user of the subprogram must know the names of formal parameters."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 613)

---

### Interacción con parámetros por defecto

Los lenguajes que admiten parámetros con palabra clave suelen combinarlos con **parámetros con valores por defecto** (*default parameters*). En ese caso, la restricción de orden se aplica de manera equivalente: en C++ (sin soporte de keyword parameters), los parámetros con valor por defecto deben aparecer todos al final de la lista formal, porque la asociación es siempre posicional.
