## Pregunta 21
> Investigue las reglas de coerción que aplica JavaScript sobre expresiones mixtas con los operadores `+`, `-`, `*`, `/`, `%` y `==`. Muestre ejemplos. ¿Cómo impactan estas decisiones en el chequeo de tipos? ¿Qué ventajas y desventajas puede apreciar?

---

## Respuesta

### 1. Coerción en JavaScript

En JavaScript, cuando los operandos de una expresión mixta no son del mismo tipo, el lenguaje aplica **coerción implícita**: convierte automáticamente uno o ambos operandos al tipo que la operación requiere, evitando lanzar un error de tipos en la mayoría de los casos.

> *"Un sistema de tipos flexible brinda reglas de conversión o coerción que permiten aceptar datos de un tipo Q en un contexto en el cual se espera un dato de tipo T. La conversión puede ser con pérdida (limitante) o sin pérdida (expansora)."*
> — 06-SistemaDeTipos (p. 2)

Este comportamiento hace que el sistema de tipos de JavaScript sea de **baja dureza respecto a expresiones mixtas**: la gran mayoría de las combinaciones de tipos con cualquier operador son aceptadas, transformando los operandos en lugar de rechazar la expresión.

> *"El objetivo es evitar que el programa se detenga por un error de tipos, para lo cual los operandos mutan hacia el tipo de dato que manda el contexto."*
> — Slides - Sistema de tipos - Python y TypeScript (p. 38)

### 2. Reglas por operador

#### Operador `+`

El `+` es el más complejo porque sirve tanto para suma aritmética como para concatenación de strings. La regla general es:

- Si **alguno de los operandos es string** (o puede convertirse a string), **el otro se convierte a string** y se realiza concatenación.
- Si ambos operandos son numéricos (o booleanos), se realiza suma aritmética.

> *Las conversiones aplicadas por JavaScript para el operador `+` cuando intervienen string y número producen una coerción número → string (concatenación).*
> — Slides - Sistema de tipos - Python y TypeScript (p. 38)

**Ejemplos:**

```javascript
"5" + 1          // ⇒ "51"    (número → string, concatena)
"Hola" + 1       // ⇒ "Hola1"
"Hola" + false   // ⇒ "Holafalse"   (boolean → string)
1 + true         // ⇒ 2       (boolean → número: true = 1)
1 + false        // ⇒ 1       (boolean → número: false = 0)
null + 1         // ⇒ 1       (null → número: null = 0)
undefined + 1    // ⇒ NaN     (undefined → NaN)
NaN + "hola"     // ⇒ "NaNhola"
[1,2,3] + "5"    // ⇒ "1,2,35"  (array → string via toString)
[1,2,3] + [4,5,6]// ⇒ "1,2,34,5,6"
```

---

#### Operadores `-`, `*`, `/`, `%`

Estos operadores son **exclusivamente aritméticos**. JavaScript convierte cualquier operando no numérico **al tipo number** antes de operar.

> *Para los operadores `-`, `*`, `/` y `%`, cuando uno de los operandos es string, JavaScript aplica la coerción string → número.*
> — Slides - Sistema de tipos - Python y TypeScript (p. 38)

**Ejemplos:**

```javascript
"42" - 1     // ⇒ 41      (string → número)
"6" * 2      // ⇒ 12
"10" / 2     // ⇒ 5
"10" % 3     // ⇒ 1
true - 1     // ⇒ 0       (true → 1)
false * 5    // ⇒ 0       (false → 0)
null - 1     // ⇒ -1      (null → 0)
undefined - 1// ⇒ NaN     (undefined → NaN)
"hola" - 1   // ⇒ NaN     (string no numérico → NaN)
"hola" * 2   // ⇒ NaN
[3] * 2      // ⇒ 6       (array de un elemento → número)
```

Una implicación importante: si el string no puede convertirse a número, el resultado es `NaN` — lo cual es un valor válido de tipo `number` en JavaScript, no un error.

---

#### Operador `==` (igualdad abstracta)

El operador `==` aplica las reglas de **Abstract Equality Comparison** de la especificación ECMAScript, que incluyen coerciones entre tipos distintos:

> *`null` y `undefined` son considerados iguales entre sí mediante `==`, y no son iguales a ningún otro valor.*
> — Slides - Sistema de tipos - Python y TypeScript (p. 38)

**Ejemplos:**

```javascript
null == undefined   // ⇒ true   (regla especial)
null == 0           // ⇒ false  (null solo iguala a null/undefined)
"0" == 0            // ⇒ true   (string → número: "0" → 0)
false == 0          // ⇒ true   (boolean → número: false → 0)
true == 1           // ⇒ true   (boolean → número: true → 1)
"" == 0             // ⇒ true   (string "" → número 0)
[] == 0             // ⇒ true   (array → string → número)
[] == ""            // ⇒ true   (array → string "")
[] == false         // ⇒ true   (cadena de coerciones)
```

El operador `===` (igualdad estricta) **no aplica coerciones**: dos valores de distinto tipo siempre son desiguales con `===`, lo que representa un comportamiento más duro.

### 3. Impacto en el chequeo de tipos

Las coerciones implícitas de JavaScript tienen un impacto directo y negativo sobre la **dureza del sistema de tipos respecto a expresiones mixtas**:

> *"Un lenguaje más duro exige que el programador establezca qué operación es la que quiere aplicar, mientras que en uno más débil las transformaciones de tipo ocurren de manera implícita para poder aplicar el operador sobre tipos no esperados sobre la misma."*
> — 06-SistemaDeTipos (p. 2)

En JavaScript, el chequeo de tipos es prácticamente nulo para expresiones con operadores aritméticos y `==`: el intérprete raramente emite errores de tipo, pues convierte los operandos antes de operar. Esto implica:

- Los **errores de tipo pasan silenciosamente** como valores inesperados (`NaN`, strings donde se esperaba un número, etc.).
- La **detección de errores se traslada al programador**: el programa puede producir resultados incorrectos sin ningún aviso.
- El sistema de tipos **no puede garantizar type safety** — JavaScript no es un lenguaje fuertemente tipado.

> *"Las conversiones limitantes se consideran inseguras, porque se pierde información que no puede recuperarse."*
> — 06-SistemaDeTipos (p. 2)

Varios de los ejemplos anteriores son **conversiones limitantes** (e.g., convertir `"hola"` a número produce `NaN`, con pérdida total de información).

---

### 4. Ventajas

- **Flexibilidad y rapidez de prototipado**: el programador no necesita convertir tipos manualmente para operaciones simples.
- **Tolerancia a entradas variables**: por ejemplo, datos numéricos provenientes de formularios web son strings; con coerciones, operaciones como `"10" * 2` funcionan sin conversión explícita.
- **Menor verbosidad**: el código resulta más conciso en escenarios donde los tipos son predecibles.

---

### 5. Desventajas

- **Errores silenciosos difíciles de depurar**: expresiones como `"5" + 3` producen `"53"` en lugar de `8`, lo que puede provocar bugs sutiles que no generan ninguna excepción.
- **Comportamientos contraintuitivos**: las reglas de `==` son especialmente confusas (`[] == false`, `"" == 0`, etc.), lo que dificulta razonar sobre el código.
- **Baja seguridad del sistema de tipos**: al no detectar errores de tipo, se compromete la confiabilidad del programa.
- **Interacción problemática con sobrecarga**: la sobrecarga del operador `+` (suma vs. concatenación) combinada con coerciones genera ambigüedad semántica.

> *"Las coerciones implícitas, junto con la sobrecarga y las expresiones mixtas, pueden generar situaciones en las que los lenguajes están obligados a recortar el poder expresivo de alguna de las características para poder resolver satisfactoriamente cualquier expresión."*
> — 06-SistemaDeTipos (p. 4)

Precisamente por estas razones surge **TypeScript**, que mantiene la semántica de JavaScript pero agrega tipado estático y restringe algunas coerciones, elevando la dureza del sistema de tipos en tiempo de compilación.

> *"TypeScript busca mantener la semántica de JavaScript, pero hace controles para reconocer errores de tipos que en JavaScript no se reconocen."*
> — Slides - Sistema de tipos - Python y TypeScript (p. 28)

### Resumen comparativo de reglas

| Tipos encontrados        | Operador    | Coerción aplicada              | Ejemplo                      |
|--------------------------|-------------|-------------------------------|------------------------------|
| boolean + número         | cualquiera  | boolean → número               | `true + 14 ⇒ 15`             |
| string + número          | `+`         | número → string (concatena)   | `"Hola" + 1 ⇒ "Hola1"`      |
| string + número          | `-, *, /, %`| string → número (opera)       | `"42" - 1 ⇒ 41`             |
| objeto + primitivo       | cualquiera  | objeto → primitivo (valueOf)  | `{valueOf(){return 4}} + 1 ⇒ 5` |
| null / undefined         | `==`        | solo iguales entre sí         | `null == undefined ⇒ true`   |
| string + número          | `==`        | string → número               | `"0" == 0 ⇒ true`           |

> — Slides - Sistema de tipos - Python y TypeScript (p. 38)
