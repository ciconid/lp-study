## Respuesta

### 1. Marco de análisis: decisiones de diseño para arreglos

Sebesta identifica varias decisiones de diseño clave para los arreglos, entre ellas: qué tipos son válidos como índices, si se realiza chequeo de rango de los índices, cuándo se ligan los rangos del subscript, cuándo se asigna el almacenamiento, si se permiten arreglos multidimensionales, si se admiten valores iniciales y si se permiten porciones (*slices*).

> *"A heap-dynamic array is one in which the binding of subscript ranges and storage allocation is dynamic and can change any number of times during the array's lifetime. The advantage of heap-dynamic arrays over the others is flexibility: Arrays can grow and shrink during program execution as the need for space changes."*
> — Sebesta, Concepts of Programming Languages (p. 424)

> *"In Java, all non-generic arrays are fixed heap-dynamic. Once created, these arrays keep the same subscript ranges and storage."*
> — Sebesta, Concepts of Programming Languages (p. 425)

### 2. Arreglos en Java

Java tiene **dos mecanismos** para trabajar con arreglos: los arreglos nativos (`T[]`) y la clase genérica `ArrayList<T>`.

#### 2.1 Arreglos nativos (`T[]`)

- **Homogéneos**: todos los elementos deben ser del mismo tipo declarado estáticamente.
- **Fixed heap-dynamic**: el tamaño se fija al crear el arreglo con `new` y **no puede cambiar** posteriormente.
- **Índices**: siempre desde `0` hasta `length - 1`. No admite índices negativos.
- **Chequeo de rango**: en tiempo de ejecución; un acceso fuera de rango lanza `ArrayIndexOutOfBoundsException`.
- **Sin slices nativos**: no hay sintaxis de porción; se usa `Arrays.copyOfRange()`.
- **Multidimensional**: soporta arreglos de arreglos (`int[][]`), que pueden ser de forma irregular (*jagged arrays*).

```java
// Declaración e inicialización
int[] nums = new int[5];         // tamaño fijo: 5 enteros
int[] primes = {2, 3, 5, 7, 11}; // inicialización directa

// Acceso y asignación
nums[0] = 10;
int x = primes[2];  // ⇒ 5

// Longitud
System.out.println(primes.length);  // ⇒ 5

// Error de rango
int y = primes[10];  // ⇒ ArrayIndexOutOfBoundsException

// Arreglo multidimensional
int[][] matrix = new int[3][4];
int[][] jagged = {{1, 2}, {3, 4, 5}, {6}};
```

#### 2.2 `ArrayList<T>` (heap-dynamic genérico)

Para comportamiento dinámico (tamaño variable), Java provee `ArrayList<T>`, que es **heap-dynamic** y tipado estáticamente con genéricos.

```java
import java.util.ArrayList;

ArrayList<String> lista = new ArrayList<>();
lista.add("Python");
lista.add("Java");
lista.add("Go");
System.out.println(lista.get(1));  // ⇒ "Java"
lista.remove(0);
System.out.println(lista.size());  // ⇒ 2
```

> *"Java includes a generic class similar to C#'s List, named ArrayList. It is different from C#'s List in that subscripting is not supported—get and set methods must be used to access elements."*
> — Sebesta, Concepts of Programming Languages (p. 425)

### 3. Listas/arreglos en Python

Python no tiene un tipo "arreglo" nativo diferenciado: sus **listas** (`list`) cumplen el rol de arreglos y son significativamente más flexibles.

> *"Python includes a list data type, which also serves as Python's arrays. Unlike the lists of Scheme, Common Lisp, ML, and F#, the lists of Python are mutable. They can contain any data value or object."*
> — Sebesta, Concepts of Programming Languages (p. 452)

- **Heterogéneas**: pueden contener elementos de tipos distintos en la misma lista.
- **Heap-dynamic**: el tamaño puede cambiar en cualquier momento (`append`, `remove`, `pop`, etc.).
- **Tipado dinámico**: no se declara el tipo de los elementos.
- **Índices**: desde `0`, pero también admite **índices negativos** (`-1` es el último elemento).
- **Slicing nativo**: `lista[inicio:fin:paso]`, sin necesidad de funciones auxiliares.
- **Chequeo de rango**: en tiempo de ejecución; un acceso fuera de rango lanza `IndexError`.
- **Operaciones ricas**: `sort()`, `reverse()`, `count()`, `extend()`, `index()`, etc.
- **List comprehensions**: sintaxis concisa para construir listas a partir de otras.

```python
# Heterogénea y heap-dynamic
lista = [3, 5.8, "grape"]
print(lista[1])    # ⇒ 5.8
print(lista[-1])   # ⇒ "grape"  (índice negativo)

# Slicing
nums = [0, 1, 2, 3, 4, 5]
print(nums[1:4])   # ⇒ [1, 2, 3]
print(nums[::2])   # ⇒ [0, 2, 4]  (paso 2)

# Tamaño dinámico
lista.append(True)
lista.append([1, 2])  # lista dentro de lista
print(len(lista))     # ⇒ 5

# Modificar
lista[0] = 99
lista.remove(5.8)

# List comprehension
cuadrados = [x**2 for x in range(5)]  # ⇒ [0, 1, 4, 9, 16]
pares = [x for x in range(10) if x % 2 == 0]

# Multidimensional (lista de listas)
matriz = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
print(matriz[1][2])  # ⇒ 6
```

> *"A list value is a sequence of expressions that are separated by commas and delimited with brackets. The elements of a list are indexed starting at zero. List elements also can be updated by assignment."*
> — Sebesta, Concepts of Programming Languages (p. 452)

### 4. Análisis comparativo

| Criterio                         | Java (arreglo nativo `T[]`)           | Java (`ArrayList<T>`)             | Python (`list`)                        |
|----------------------------------|---------------------------------------|-----------------------------------|----------------------------------------|
| **Homogeneidad de elementos**    | Sí — tipo declarado estáticamente     | Sí — tipo declarado con genérico  | No — heterogénea                       |
| **Categoría de binding**         | Fixed heap-dynamic                    | Heap-dynamic                      | Heap-dynamic                           |
| **Tamaño variable**              | No — fijo al crear                    | Sí                                | Sí                                     |
| **Tipo de índices**              | `int`, desde 0                        | `int`, desde 0 (via `get`)        | `int`, desde 0; negativos admitidos    |
| **Chequeo de rango**             | Sí, en ejecución (`ArrayIndexOutOfBoundsException`) | Sí (`IndexOutOfBoundsException`) | Sí, en ejecución (`IndexError`)        |
| **Slicing nativo**               | No                                    | No                                | Sí (`lista[a:b:paso]`)                 |
| **Declaración de tipo**          | Explícita (tipado estático)           | Explícita (genérico)              | Implícita (tipado dinámico)            |
| **List comprehensions**          | No                                    | No                                | Sí                                     |
| **Multidimensional**             | Sí (`int[][]`, jagged)                | No directamente                   | Sí (listas de listas)                  |

---

### 5. Impacto en el sistema de tipos

- **Java** garantiza en tiempo de compilación que todos los elementos de un arreglo nativo son del tipo declarado → mayor seguridad de tipos (mayor dureza). El uso de `ArrayList<T>` mantiene esa seguridad con flexibilidad de tamaño.
- **Python** no realiza ningún chequeo de tipos sobre los elementos en la declaración; cualquier objeto puede coexistir en la misma lista → mayor flexibilidad, menor dureza respecto al tipo de elementos.

El chequeo de rango de índices existe en ambos lenguajes (en ejecución), pero Java lo formaliza con una excepción de tipo controlada (`ArrayIndexOutOfBoundsException`), mientras que Python lanza `IndexError`.
