# Resumen Clase 4: Estructuras de Control

## Introducción
Las estructuras de control determinan el orden de ejecución de las instrucciones en un programa y son esenciales para expresar la lógica de control de flujo.

## Categorías principales
- **Secuencia:** Es la ejecución ordenada de instrucciones una tras otra, tal como aparecen en el código. Es la forma más básica de control de flujo.
- **Selección (condicionales):** Permite tomar decisiones y ejecutar diferentes bloques de código según se cumpla o no una condición. Ejemplos: if, else, switch/case.
- **Iteración (bucles):** Permite repetir un bloque de instrucciones mientras se cumpla una condición o durante un número determinado de veces. Ejemplos: for, while, do-while.
- **Abstracción procedural:** Consiste en encapsular un conjunto de instrucciones en procedimientos o funciones, facilitando la reutilización y organización del código.
- **Recursión:** Es la capacidad de una función o procedimiento de llamarse a sí mismo, permitiendo resolver problemas de manera repetitiva o dividirlos en subproblemas.
- **Concurrencia:** Permite que varias partes del programa se ejecuten de manera simultánea o intercalada, aprovechando mejor los recursos y facilitando la programación de tareas independientes.
- **Manejo de excepciones:** Proporciona mecanismos para detectar y gestionar errores o situaciones excepcionales durante la ejecución, evitando que el programa termine abruptamente.
- **No determinismo:** Permite que el flujo de ejecución no esté completamente determinado por el código, sino que pueda depender de elecciones aleatorias o externas, útil en ciertos algoritmos y modelos de concurrencia.

## Consideraciones de diseño
- Variantes en la implementación de ciclos (por ejemplo, el ciclo for puede variar entre lenguajes en cuanto a la vida de la variable de control, momento de evaluación de la condición, etc.)
- Impacto de las estructuras de control en la semántica del lenguaje

## Importancia
Comprender las opciones y variantes de las estructuras de control es fundamental para escribir programas correctos y eficientes en cualquier lenguaje de programación.
