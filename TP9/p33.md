## Pregunta 33
> Una definición del tipo árbol binario en SML puede ser la siguiente:
> ```sml
> datatype ‘a arbolb = hoja of ‘a | nodo of ‘a arbolb * ‘a * ‘a arbolb;
> ```
> Y si queremos que los elementos en las hojas puedan ser de un tipo diferente al de los nodos internos, podemos utilizar la siguiente definición:
> ```sml
> datatype (‘a, ‘b) arbolb = hoja of ‘b | nodo of (((‘a, ‘b) arbolb) * ‘a * ((‘a, ‘b) arbolb));
> ```
> Escriba la expresión de tipos para una función que, dado un árbol, genere una lista con todos los elementos almacenados en las hojas del árbol. ¿Sería posible generar una lista con todos los elementos y en las hojas y en los nodos internos de este árbol?

### Respuesta

#### 1. Expresión de tipos para una función que lista los elementos de las hojas
Para el primer tipo:
```sml
hojas : 'a arbolb -> 'a list
```
Para el segundo tipo:
```sml
hojas : ('a, 'b) arbolb -> 'b list
```

#### 2. ¿Es posible generar una lista con todos los elementos (hojas y nodos internos)?
Para el primer tipo, sí:
```sml
todos : 'a arbolb -> 'a list
```
La función puede recorrer el árbol y recolectar todos los valores (de hojas y nodos) en una lista.

Para el segundo tipo, no es posible una lista homogénea, ya que las hojas y los nodos internos pueden tener tipos distintos (`'a` y `'b`). Se podría generar una lista de pares o una estructura que distinga el origen de cada elemento, o bien usar un tipo unión (sum type) si el lenguaje lo permite.

> *"En SML, los tipos de datos polimórficos permiten definir árboles cuyos nodos y hojas pueden almacenar distintos tipos de valores. Si los tipos son diferentes, no es posible generar una lista homogénea con todos los elementos."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 49)
