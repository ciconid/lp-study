## Pregunta 24
> Compare la dureza del sistema de tipos respecto a las expresiones mixtas de Haskell con la de Pascal y la de Python.

### Respuesta

La dureza del sistema de tipos respecto a las expresiones mixtas varía significativamente entre Haskell, Pascal y Python.

**Haskell**

Haskell no soporta expresiones mixtas ni incluye coerciones automáticas. Cualquier operación que involucre tipos diferentes requiere que el programador realice una conversión explícita utilizando funciones provistas por el lenguaje (por ejemplo, `fromIntegral` para convertir un entero a un flotante). Si se intenta operar entre tipos distintos sin conversión, se produce un error de tipos.
> *"El lenguaje no soporta expresiones mixtas ni incluye coerciones. Cualquier operación que involucre tipos diferentes requiere que el programador realice una conversión explícita utilizando funciones provistas por el lenguaje (como fromIntegral para pasar de tipos enteros a tipos de punto flotante)."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 31)

**Pascal**

Pascal permite expresiones mixtas numéricas, realizando conversiones implícitas entre tipos numéricos compatibles (por ejemplo, entre `integer` y `real`). Sin embargo, no permite mezclar tipos incompatibles (como sumar un string y un número), y en esos casos produce un error de tipos. Esto implica una dureza intermedia: hay coerciones automáticas entre tipos numéricos, pero no entre tipos incompatibles.
> *"Un lenguaje más duro exige que el programador establezca qué operación es la que quiere aplicar, mientras que en uno más débil las transformaciones de tipo ocurren de manera implícita para poder aplicar el operador sobre tipos no esperados sobre la misma."*
> — 06-SistemaDeTipos.pdf (p. 2)

**Python**

Python permite algunas expresiones mixtas con conversiones implícitas (por ejemplo, `1 + 2.0` da `3.0`, `True + 1` da `2`). Sin embargo, no permite mezclar tipos incompatibles (por ejemplo, `"hola" + 1` da error). Es decir, es flexible, pero no tanto como JavaScript; hay coerciones automáticas en muchos casos, pero no en todos.
> *"Pese a las conversiones consideradas, no cualquier combinación de tipos son permitidas en el contexto de las expresiones."*
> — Slides - Sistema de tipos - Python y TypeScript (p. 17)

**Resumen comparativo:**
- Haskell: sistema más duro, exige conversiones explícitas.
- Pascal: dureza intermedia, coerciones automáticas entre tipos numéricos compatibles.
- Python: sistema más flexible, coerciones automáticas en muchos casos, pero no permite todas las combinaciones.

> *"Podemos analizar el grado de dureza del sistema tipos con respecto a las expresiones mixtas: Lenguaje con Tipado menos duro [...] Lenguaje con Tipado más duro [...] Un lenguaje más duro exige que el programador establezca qué operación es la que quiere aplicar, mientras que en uno más débil las transformaciones de tipo ocurren de manera implícita para poder aplicar el operador sobre tipos no esperados sobre la misma."*
> — 06-SistemaDeTipos.pdf (p. 2)
