## Pregunta 34
> Dada la siguiente secuencia de declaraciones en SML:
> ```sml
> fun por2 x = x*2
> val funcion1 = map por2;
> fun funcion2 lista = map por2 lista;
> ```
> ¿Cuáles son las diferencias y similitudes entre funcion1 y funcion2?

### Respuesta

- **Similitudes:**
  - Ambas aplican la función `por2` a todos los elementos de una lista, devolviendo una lista con los resultados.
  - Ambas pueden usarse para duplicar los elementos de una lista de enteros.
  - El tipo de ambas es `int list -> int list`.

- **Diferencias:**
  - `funcion1` es una función definida por aplicación parcial: `map por2` retorna una función que espera una lista como argumento.
  - `funcion2` es una función definida explícitamente con un parámetro, pero su comportamiento es idéntico a `funcion1`.
  - En la práctica, ambas son equivalentes en uso y eficiencia, pero `funcion1` es más concisa y aprovecha la aplicación parcial.

> *"La aplicación parcial permite definir funciones especializadas de manera concisa. Definir una función como 'val f = map g' es equivalente a 'fun f xs = map g xs', pero la primera es más idiomática en lenguajes funcionales."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 16, 35)
