## Pregunta 14
> ¿Python tiene semántica de valores para sus variables? Justifique si su respuesta fue afirmativa o muestre un contra-ejemplo en caso contrario.

---

### Respuesta

Python no tiene semántica de valores para sus variables, sino semántica de referencias. En Python, las variables son nombres que se ligan a objetos, y estos objetos pueden ser mutables o inmutables. Cuando se asigna una variable a otra, ambas referencian el mismo objeto (no se copia el valor, salvo para tipos inmutables donde el comportamiento es similar a una copia por valor).

Por ejemplo:
```python
x = [1, 2, 3]
y = x
y.append(4)
print(x)  # [1, 2, 3, 4]
```
En este caso, tanto `x` como `y` referencian la misma lista. Modificar `y` afecta a `x`, lo que demuestra que la semántica es de referencia y no de valor.

> *"El aliasing es un concepto que viene asociado a la interpretación de la variable como abstracción de lo almacenado en una celda de memoria."*
> — 02-Nombres-Alcance-Ligaduras.pdf (p. 2)

En cambio, para tipos inmutables como los enteros:
```python
x = 5
y = x
y = 7
print(x)  # 5
```
Aquí, `y` pasa a referenciar un nuevo objeto, y `x` no se ve afectado. Sin embargo, esto no implica semántica de valores, sino que los objetos inmutables no pueden ser modificados.

---

#### Información adicional
La semántica de referencia de Python es fundamental para entender el comportamiento de las variables y evitar errores sutiles, especialmente con objetos mutables.

> *"In Python, variables are references to objects, not containers of values. Assignment binds a name to an object."*
> — Python Documentation, Data model (docs.python.org)
