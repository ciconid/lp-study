## Pregunta 21
> Utilice algún iterador para recorrer el objeto receptor en cada uno de los siguientes incisos.

---

### a. `Array >> calcular:` — similar a `collect:`

Retorna un arreglo donde cada elemento es el resultado de evaluar el bloque para el elemento correspondiente del receptor.

```smalltalk
Array extend [
    calcular: aBlock [
        | resultado i |
        resultado := Array new: self size.
        i := 1.
        self do: [:elem |
            resultado at: i put: (aBlock value: elem).
            i := i + 1.
        ].
        ^resultado
    ]
]
```

**Verificación:**
```smalltalk
#(4 1 2 3) calcular: [:i | i even.]         "→ #(true false true false)"
#(4 1 2 3) calcular: [:i | i + 10.]         "→ #(14 11 12 13)"
#('hola' 1 'no' 3.0) calcular: [:i | i isInteger.]  "→ #(false true false false)"
```

> *"La clase Collection provee iteradores como `do:`, `collect:`, `select:`, `reject:`."*
> — Slides - POO y Smalltalk.pdf (p. 22)

**Nota:** La implementación es equivalente a `collect:` de la clase `Collection`. Se usa `do:` como iterador base para recorrer el receptor.

---

### b. `Collection >> incluye:` — similar a `includes:`

Determina si el receptor contiene algún elemento igual al argumento recibido.

```smalltalk
Collection extend [
    incluye: anObject [
        self do: [:elem |
            elem = anObject ifTrue: [^true].
        ].
        ^false
    ]
]
```

**Verificación:**
```smalltalk
#(3 'si' 1 2) incluye: 'si'        "→ true"
'murcielago' incluye: $u            "→ true"
#(2.0 'no' 2 $a) incluye: 4        "→ false"
```

**Notas:**
- Se extiende `Collection` (en lugar de `Array`) para que el método funcione con cualquier colección, incluyendo `String` (cuyos elementos son `Character`).
- El método termina anticipadamente (`^true`) en cuanto encuentra el elemento, implementando así una forma de **evaluación no estricta / corto-circuito** para la búsqueda.
- `=` en Smalltalk compara por valor (equivalencia estructural), no por identidad.

---

### c. `String >> soloLetras`

Retorna un nuevo `String` con solo las letras presentes en la cadena receptora.

```smalltalk
String extend [
    soloLetras [
        | resultado |
        resultado := ''.
        self do: [:c |
            c isLetter ifTrue: [resultado := resultado , c asString].
        ].
        ^resultado
    ]
]
```

**Verificación:**
```smalltalk
'Tengo 30 anios' soloLetras   "→ 'Tengoanios'"
```

**Notas:**
- `do:` itera sobre los `Character` que forman el `String`.
- `isLetter` devuelve `true` si el carácter es una letra (alfabético).
- `c asString` convierte el `Character` a `String` de un carácter para poder concatenarlo con `,`.
- `resultado := resultado , c asString` construye el resultado de forma acumulativa.

---

### Consideración común a los tres incisos

Los tres métodos siguen el mismo patrón: recorrer el receptor con `do:` (iterador base de `Collection`) y acumular o verificar dentro del bloque. Esto es exactamente el principio de los iteradores en Smalltalk: la colección controla el recorrido y delega la lógica al bloque recibido como argumento.
