## Pregunta 32
> Explique de qué forma difiere el proceso de inferencia de tipos en Haskell en comparación con SML. Indique las principales razones detrás de esas diferencias. Ilustre con ejemplos a partir de las funciones desarrolladas en los ejercicios 25 y 26.

### Respuesta

#### Inferencia de tipos en Haskell
Haskell utiliza un sistema de inferencia de tipos basado en el algoritmo Hindley-Milner, pero con extensiones que permiten mayor polimorfismo y uso de clases de tipos. El compilador infiere el tipo más general posible para cada función, agregando restricciones de clase cuando es necesario.

> *"Una parte fundamental del lenguaje es su mecanismo de inferencia de tipos. El tipo asociado a un identificador puede ser inferido a partir del contexto en el que aparece el identificador."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 32)

Por ejemplo, en Haskell:
```haskell
id x = x
-- id :: a -> a

suma x y = x + y
-- suma :: Num a => a -> a -> a
```
El compilador agrega la restricción `Num a` porque usa el operador `+`.

#### Inferencia de tipos en SML
SML también utiliza Hindley-Milner, pero no tiene clases de tipos. La inferencia es puramente paramétrica: si una función usa `+`, el tipo será concreto (`int -> int -> int`), ya que no hay clases de tipos para generalizar sobre operaciones.

Por ejemplo, en SML:
```sml
fun id x = x;
(* id : 'a -> 'a *)

fun suma x y = x + y;
(* suma : int -> int -> int *)
```

#### Diferencias principales
- **Clases de tipos:** Haskell permite polimorfismo restringido (por clases de tipos), SML solo polimorfismo paramétrico.
- **Generalización:** Haskell infiere el tipo más general compatible con las restricciones de clase; SML infiere el tipo más general posible, pero sin restricciones de clase.
- **Operadores:** En Haskell, operadores como `+` pueden usarse con cualquier tipo que sea instancia de `Num`. En SML, `+` solo está definido para `int` o `real`.

#### Ejemplo con funciones de los ejercicios 25 y 26
- En Haskell:
```haskell
apply f x = f x
-- apply :: (a -> b) -> a -> b

suma [] = 0
suma (x:xs) = x + suma xs
-- suma :: Num a => [a] -> a
```
- En SML:
```sml
fun apply f x = f x;
(* apply : ('a -> 'b) -> 'a -> 'b *)

fun suma [] = 0
  | suma (x::xs) = x + suma xs;
(* suma : int list -> int *)
```

> *"El compilador nunca elige un tipo concreto (como Int o Float) si no hay nada en el código que lo fuerce a hacerlo. Siempre va a preferir la versión polimórfica restringida."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 33)

**En resumen:**
- Haskell permite mayor polimorfismo gracias a las clases de tipos.
- SML es más restrictivo: la inferencia es paramétrica y los operadores solo funcionan con tipos concretos.
