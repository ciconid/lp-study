## Pregunta 28
> Compare los mecanismos para definir nuevas funciones como aplicaciones parciales de Python y SML. Utilice al menos dos criterios de los estudiados para realizar la comparación.

### Respuesta

#### SML
En SML, las funciones curried permiten definir nuevas funciones por aplicación parcial de manera directa. Por ejemplo:
```sml
fun sumar x y = x + y;
val sumarCinco = sumar 5;
```
Aquí, `sumarCinco` es una nueva función que suma 5 a su argumento. El mecanismo es nativo y transparente, gracias al soporte de funciones curried.
> *"Las funciones deﬁnidas de manera curried van a permitir realizar aplicaciones parciales."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 46)

#### Python
En Python, las funciones no son curried por defecto. Para lograr aplicaciones parciales, se debe usar herramientas como `functools.partial`:
```python
from functools import partial

def sumar(x, y):
    return x + y

sumar_cinco = partial(sumar, 5)
```
Aquí, `sumar_cinco` es una función que suma 5 a su argumento, pero requiere una librería estándar y no es nativo del lenguaje.
> *"En Python, para realizar aplicaciones parciales se utiliza la función partial del módulo functools, ya que las funciones no son curried por defecto."*
> — Slides - Sistema de tipos - Python y TypeScript (p. 18)

#### Comparación por criterios
- **Simplicidad sintáctica:** En SML, la aplicación parcial es directa y natural gracias al currying. En Python, requiere importar y usar una función adicional.
- **Transparencia de tipos:** En SML, el sistema de tipos infiere correctamente el tipo de la función parcial. En Python, el tipado es dinámico y no se verifica hasta tiempo de ejecución.

> *"El currying permite crear funciones a partir de la aplicación parcial de otras funciones."*
> — Slides - Paradigma Funcional - Haskell y SML (p. 16)

**Conclusión:**
SML ofrece un mecanismo más simple y seguro para aplicaciones parciales, mientras que en Python es posible pero menos directo y con menor control de tipos.
