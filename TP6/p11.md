## Pregunta 11
> ¿Qué característica presenta `def` en Python que lo distingue de los mecanismos de definición de funciones en otros lenguajes? Investigue el uso del bloque `let ...in ...end` en SML, y determine si el lenguaje tiene el mismo comportamiento que Python.

---

## Respuesta

### La característica distintiva de `def` en Python

En la mayoría de los lenguajes de programación, la definición de funciones es una **declaración estática** que el compilador procesa en tiempo de compilación y que está sintácticamente restringida a ubicaciones fijas en el código (generalmente al nivel de módulo o clase).

Python rompe con esta convención: **`def` es una instrucción ejecutable en tiempo de ejecución**.

> *"One characteristic of Python functions that sets them apart from the functions of other common programming languages is that function def statements are executable. When a def statement is executed, it assigns the given name to the given function body. Until a function's def has been executed, the function cannot be called."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 609–610)

Esto implica que:

1. **Una función no existe hasta que su `def` es ejecutado**: a diferencia de lenguajes como C, C++ o Java donde las funciones/métodos son visibles en toda su unidad de compilación, en Python la función solo puede llamarse después de que la instrucción `def` fue ejecutada en el flujo de control.

2. **`def` puede aparecer en cualquier lugar donde puede aparecer una instrucción**: dentro de `if/else`, dentro de bucles, en el cuerpo de otra función, etc.

3. **Permite definir distintas versiones de una función según el flujo de control en tiempo de ejecución**:

```python
if condicion:
    def fun(x):
        return x + 1
else:
    def fun(x):
        return x * 2
```

Solo una de las dos versiones de `fun` estará disponible, dependiendo de qué rama se ejecute.

4. **Crea cierres (*closures*)**: cuando `def` aparece dentro de otra función, la función definida captura el ambiente léxico en el momento de su ejecución, accediendo a las variables del ámbito externo:

```python
def make_adder(n):
    def adder(x):
        return x + n   # 'n' es capturado del ámbito externo
    return adder

add5 = make_adder(5)
print(add5(3))  # 8
```

### El bloque `let ...in ...end` en SML

En SML (*Standard ML*), el equivalente funcional para definir funciones locales es el bloque `let...in...end`. Su estructura es:

```sml
let
  val nombre = valor
  fun f parametros = expresion
in
  expresion_que_usa_nombre_y_f
end
```

Las definiciones en la parte `let` son **visibles únicamente dentro del bloque `in...end`**. Esto establece un alcance léxico local, análogo al de una función Python interna.

**Ejemplo concreto** (del libro de texto):

```sml
let val radius = 2.7
    val pi = 3.14159
in pi * radius * radius
end;
```

> *"The normal use of val is in a let expression [...] val declarations are often nested in let constructs in ML."*
> — Sebesta, *Concepts of Programming Languages* E12 (p. 1044, 538)

Con funciones:

```sml
let fun square x = x * x
in square 5 + square 3
end;
```

La función `square` solo existe dentro del `in...end`.

### ¿Tiene SML el mismo comportamiento que Python?

**Similitudes:**

| Aspecto | Python (`def` anidado) | SML (`let fun`) |
|---------|----------------------|-----------------|
| Alcance | Léxico (estático) | Léxico (estático) |
| Cierre | Sí: captura el ambiente externo | Sí: captura el ambiente del `let` |
| Localidad | La función es local al bloque/función que la contiene | La función es local al `in...end` |

En ambos casos, las funciones locales crean cierres sobre el ambiente de su definición y solo son accesibles dentro de su alcance léxico.

**Diferencias clave:**

| Aspecto | Python (`def`) | SML (`let...in...end`) |
|---------|---------------|------------------------|
| Naturaleza | **Instrucción ejecutable** (imperativa) | **Expresión funcional** (no es una instrucción) |
| Cuándo se crea el enlace | En tiempo de ejecución, cuando se alcanza el `def` | Cuando se evalúa la expresión `let` |
| Condicionalidad | Puede estar dentro de `if/else`, loops, etc. → definición condicional posible | Siempre se crean todos los enlaces del `let` al evaluar la expresión |
| Efecto en el ámbito | Agrega un nombre al ámbito local de la función envolvente | Crea un ámbito nuevo que solo existe en el `in...end` |
| Paradigma | Imperativo: `def` es una instrucción con efecto lateral (modificar el namespace) | Funcional: `let` es una expresión que evalúa a un valor |

**Conclusión:** SML y Python comparten el soporte a funciones locales con cierre léxico, pero difieren en el mecanismo subyacente. En Python, `def` es una instrucción con efecto lateral dinámico que puede ser condicionalmente ejecutada. En SML, `let...in...end` es una expresión funcional pura que crea un ámbito local de manera determinista al momento de su evaluación. SML **no** tiene el comportamiento de poder definir condicionalmente diferentes versiones de una función en tiempo de ejecución de la manera que Python lo hace con `def` dentro de un `if/else`.

### Información adicional

La flexibilidad de `def` en Python refleja que en Python **las funciones son objetos de primera clase**: `def` simplemente enlaza un nombre a un objeto función en el espacio de nombres local, de la misma manera en que una asignación `x = 5` enlaza un nombre a un entero. En SML, las funciones también son valores de primera clase, pero el lenguaje es puramente funcional, por lo que no existe el concepto de "modificar un espacio de nombres" como efecto lateral de una declaración.
