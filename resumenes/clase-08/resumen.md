# Resumen Clase 8: Encapsulamiento y Abstracción

## Introducción
El encapsulamiento y la abstracción permiten ocultar detalles de implementación y exponer solo lo necesario, facilitando la modularidad y el mantenimiento del software.

## Tipos de abstracción
- **Tipo de dato abstracto (TDA):** Definido por el usuario, oculta la representación interna y expone operaciones públicas.
- **Encapsulamiento:** El lenguaje prohíbe el acceso directo a la información oculta.
- **Ocultamiento de información:** El usuario no necesita conocer los detalles internos para usar la abstracción.

## Implementación orientada a objetos
- Clases, herencia, métodos y atributos
- Registro de activación para métodos
- Estrategias de herencia: delegación y copia
- Identidad de objetos y atributos de instancia

## Importancia
El encapsulamiento y la abstracción son fundamentales para el diseño de software robusto, permitiendo dividir el código en módulos independientes y reutilizables.
